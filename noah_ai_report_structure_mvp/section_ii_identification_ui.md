# AI Report - Section II: Patient Identification (UI Blueprint)

This document outlines the conceptual UI structure for Section II of the AI-generated shift report.
It shows where data generated by `populate_section_ii_patient_identification(patient_id)` would be displayed for a specific patient.

```html
<!-- Conceptual HTML-like structure for a single patient card -->
<div class="section-ii-patient-identification">
  <div class="header">
    <h3>Section II: Patient Identification</h3>
    <span class="room-number">Room: {{ room_no }}</span> <!-- Populated by: room_no -->
  </div>

  <div class="patient-details-grid">
    <div class="detail-item patient-banner">
      <strong>Patient:</strong> <span id="patient-name-dob-mrn">{{ patient_name_dob_mrn }}</span> <!-- Populated by: patient_name_dob_mrn -->
    </div>

    <div class="detail-item">
      <strong>Isolation Precautions:</strong>
      <span id="isolation-precautions" class="status-{{ isolation_precautions | lower | replace(' ', '-') }}">{{ isolation_precautions }}</span> <!-- Populated by: isolation_precautions -->
      <!-- Clicking could show details or allow editing if permissions allow -->
    </div>

    <div class="detail-item">
      <strong>Code Status:</strong>
      <span id="code-status" class="status-{{ code_status | lower | replace('/', '-') }}">{{ code_status }}</span> <!-- Populated by: code_status -->
      <!-- Clicking could show details or allow editing -->
    </div>

    <div class="detail-item wide">
      <strong>Allergies:</strong> <span id="allergies">{{ allergies }}</span> <!-- Populated by: allergies -->
      <!-- Clicking could show details or allow adding new allergy -->
    </div>

    <div class="detail-item wide">
      <strong>Admitted On / Principal Problem:</strong> <span id="admitted-on-problem">{{ admitted_on_problem }}</span> <!-- Populated by: admitted_on_problem -->
    </div>

    <div class="detail-item wide">
      <strong>Consulting Physicians:</strong> <span id="consulting-physicians">{{ consulting_physicians }}</span> <!-- Populated by: consulting_physicians -->
      <!-- Could list each physician with contact info or specialty -->
    </div>

    <div class="detail-item">
      <strong>Emergency Contact:</strong> <span id="emergency-contact">{{ emergency_contact }}</span> <!-- Populated by: emergency_contact -->
    </div>
  </div>
</div>

- code_status -> {{ code_status }} (dynamic class for styling based on status)
- allergies -> {{ allergies }}

- consulting_physicians -> {{ consulting_physicians }}
- emergency_contact -> {{ emergency_contact }}

This section would typically be part of a larger patient-specific view or repeated for each patient in a list.
```
